# 模块开发说明

本文档说明队友拿到统一后端骨架后如何开始各自模块开发。

## 1. 不允许做的事

1. 不允许单独新建后端项目。
2. 不允许私自新增公共表。
3. 不允许在 `student`、`teacher` 表中保存密码。
4. 不允许考勤模块实现文档提交和文档审核。
5. 不允许成绩评估与过程分析模块修改原始考勤、过程文档、开发日志和选题申请数据。
6. 不允许提交本地数据库密码、API Key、GitHub Token。

## 2. 各模块开发目录

| 模块 | 目录 |
|---|---|
| 出题管理模块 | `backend/src/main/java/com/training/system/topic` |
| 选题管理模块 | `backend/src/main/java/com/training/system/selection` |
| 学生考勤管理模块 | `backend/src/main/java/com/training/system/attendance` |
| 信息管理模块 | `backend/src/main/java/com/training/system/info` |
| 成绩评估与过程分析模块 | `backend/src/main/java/com/training/system/score` |

每个模块目录下已有：

```text
controller/
dto/
entity/
mapper/
service/
service/impl/
vo/
```

## 3. 类命名约定

| 类型 | 示例 |
|---|---|
| Controller | `TopicController` |
| Service | `TopicService` |
| ServiceImpl | `TopicServiceImpl` |
| Mapper | `TopicMapper` |
| Entity | `ProjectTopic` |
| DTO | `TopicCreateDTO` |
| VO | `TopicDetailVO` |

## 4. 接口返回

所有接口必须返回：

```java
Result<T>
```

分页接口返回：

```java
Result<PageResult<T>>
```

## 5. 数据库文件

| 文件 | 作用 |
|---|---|
| `database/sql/schema-v1.sql` | 建表 SQL |
| `database/sql/seed-v1.sql` | 测试数据 |
| `database/docs/data-dictionary.md` | 状态码、外键和字段说明 |

## 6. 提交前自查

1. 项目可以编译。
2. 接口路径符合接口规范。
3. 返回格式统一。
4. 没有修改他人模块。
5. 没有提交敏感配置。
6. 如果修改数据库，已经同步数据字典并通知负责人。
