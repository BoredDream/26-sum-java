# 接口规范制定

本文档用于统一《基于 Java 的综合实训选题系统》的后端接口设计方式。所有模块接口必须遵守统一路径、统一请求方式、统一返回格式、统一分页格式和统一权限校验规则。

## 1. 基本原则

| 原则 | 说明 |
|---|---|
| REST 风格 | 使用 HTTP 方法表达操作类型 |
| JSON 交互 | 请求和响应统一使用 JSON |
| 统一返回 | 所有接口返回统一 `Result` 格式 |
| 统一分页 | 所有列表分页返回统一分页结构 |
| 后端鉴权 | 前端隐藏按钮不能代替后端权限判断 |
| 模块边界清晰 | 不跨模块直接修改其他模块原始数据 |

## 3. HTTP 方法约定

| 操作 | 方法 | 示例 |
|---|---|---|
| 新增 | `POST` | `POST /api/topic` |
| 修改 | `PUT` | `PUT /api/topic/{topicId}` |
| 删除 | `DELETE` | `DELETE /api/topic/{topicId}` |
| 详情 | `GET` | `GET /api/topic/{topicId}` |
| 分页查询 | `GET` | `GET /api/topic/page` |
| 审核 | `POST` | `POST /api/topic/{topicId}/review` |
| 提交 | `POST` | `POST /api/topic/{topicId}/submit` |

## 4. 统一返回格式

所有接口统一返回：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

失败示例：

```json
{
  "code": 403,
  "message": "无操作权限",
  "data": null
}
```

## 5. 通用状态码

| code | 说明 |
|---|---|
| 200 | 操作成功 |
| 400 | 参数错误 |
| 401 | 未登录 |
| 403 | 无权限 |
| 404 | 数据不存在 |
| 409 | 数据冲突或重复操作 |
| 500 | 系统异常 |

## 6. 统一分页格式

分页查询请求参数建议统一为：

| 参数 | 类型 | 是否必填 | 说明 |
|---|---|---|---|
| pageNum | int | 否 | 页码，默认 1 |
| pageSize | int | 否 | 每页数量，默认 10 |
| keyword | string | 否 | 搜索关键词 |

分页响应格式：

```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "records": [],
    "total": 100,
    "pageNum": 1,
    "pageSize": 10
  }
}
```

## 7. DTO 与 VO 命名规范

| 类型 | 命名示例 | 用途 |
|---|---|---|
| 新增 DTO | `TopicCreateDTO` | 接收新增请求参数 |
| 修改 DTO | `TopicUpdateDTO` | 接收修改请求参数 |
| 查询 DTO | `TopicQueryDTO` | 接收查询条件 |
| 详情 VO | `TopicDetailVO` | 返回详情数据 |
| 列表 VO | `TopicListVO` | 返回列表数据 |

不建议直接把 Entity 返回给前端，避免暴露敏感字段或数据库内部字段。

## 8. 权限校验要求

| 场景 | 校验要求 |
|---|---|
| 学生接口 | 只能操作本人或本团队数据 |
| 教师接口 | 只能操作本人发布题目、本人指导团队、本人评分数据 |
| 管理员接口 | 可进行系统级维护，但必须记录操作日志 |
| 删除操作 | 必须校验角色和数据归属 |
| 审核操作 | 必须校验审核权限 |
| 成绩操作 | AI 只提供建议，最终成绩必须由教师确认 |

## 9. 模块接口清单模板

|---|---|---|---|---|---|---|---|
| 1 | 新增题目 | POST | `/api/topic` | `TopicCreateDTO` | `Result` | 教师 | 保存草稿或提交审核 |

## 11. 接口开发前必须确认

正式编码前，每个模块负责人必须确认：

1. 本模块接口清单是否已提交。
2. 接口路径是否和其他模块冲突。
3. 依赖的数据表是否已经进入数据库 V1。
4. 请求 DTO 和返回 VO 是否明确。
5. 权限角色是否明确。
6. 是否涉及公共表或公共状态码。
7. 是否需要其他模块提供数据。

未确认前，不建议直接写 Controller 和 Mapper。
