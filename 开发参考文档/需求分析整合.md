# 一、出题管理模块需求分析

## 1. 模块名称

出题管理模块

## 2. 模块负责人

姓名：牛滢芳

负责内容：负责出题管理模块的需求分析、页面设计、数据库表设计、功能实现与功能测试。主要包括教师出题、题目信息修改、系统资料上传、题目审核、题库管理、题目查询与筛选等内容。

## 3. 模块目标

本模块主要用于指导教师发布综合实训项目题目，并对题目信息进行统一管理。教师可以新增题目、修改本人发布的题目、上传项目相关资料，并查看题目的审核状态。管理员可以对教师提交的题目进行审核、维护题库、开放或关闭题目。学生可以查看已经审核通过并开放的题目，为后续选题提供基础信息。
1. 规范教师出题流程，避免题目信息不完整。
1. 实现题目审核机制，保证题目符合综合实训要求。
1. 集中管理项目指导书、参考资料等文件。
1. 建立题库，便于题目的查询、维护和归档。
1. 为学生选题管理模块提供准确、有效的题目信息。

## 4. 使用角色

| 管理员 | 审核题目、管理题库、开放或关闭题目、维护系统资料。 |

## 6. 功能详细说明

### 6.1 功能一：教师出题

#### 6.1 功能描述

教师登录系统后进入出题管理页面，填写综合实训项目题目信息。题目信息填写完成后，教师可以保存为草稿，也可以直接提交审核。系统应对必填项进行校验，只有题目信息填写完整后，才能提交审核。

#### 6.1 使用角色

教师

#### 6.1 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 项目名称 | 文本 | 是 | 综合实训项目题目名称 |
| 项目类型 | 文本/下拉选择 | 是 | 如应用系统、管理系统、数据分析系统等 |
| 项目难度 | 文本/下拉选择 | 是 | 如 A、B、C 或高、中、低 |
| 所需学生人数 | 数字 | 是 | 该题目允许选择的学生人数 |
| 指导教师 | 文本 | 是 | 默认当前登录教师 |
| 项目内容及要求 | 长文本 | 是 | 说明项目主要任务和完成要求 |
| 技术路线 | 长文本 | 是 | 说明开发流程、技术路线和实现方式 |
| 参考资料 | 文件/文本 | 否 | 项目指导书、任务书、参考资料等 |

#### 6.1 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 保存成功提示 | 题目保存为草稿时提示保存成功 |
| 提交成功提示 | 题目提交审核时提示提交成功 |
| 错误提示 | 必填项为空或格式错误时提示原因 |
| 题目列表 | 返回教师本人已创建的题目列表 |

#### 6.1 业务规则
1. 项目名称、项目类型、项目难度、所需学生人数、项目内容及要求、技术路线等为必填项。
1. 教师只能新增本人负责的题目，不能代替其他教师出题。
1. 题目保存为草稿后，可以继续修改。
1. 题目提交审核后，状态变为“待审核”。
1. 待审核状态下，教师不能随意修改题目。
1. 题目名称不建议完全重复，系统应进行重复提醒。
1. 所需学生人数应大于 0，且不能超过系统设定的最大人数限制。

#### 6.1 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 项目名称为空 | 提示“项目名称不能为空” |
| 所需学生人数为空或小于1 | 提示“请输入正确的学生人数” |
| 项目内容为空 | 提示“项目内容及要求不能为空” |
| 题目名称重复 | 提示“系统中已存在相似题目，请确认后再提交” |
| 当前用户不是教师 | 提示“无出题权限” |

### 6.2 功能二：题目修改

#### 6.2 功能描述

教师可以对本人创建的题目进行修改。草稿状态和被退回修改状态的题目允许修改；已审核通过或已开放选题的题目，不允许教师随意修改核心内容。

#### 6.2 使用角色

教师/管理员

#### 6.2 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 题目编号 | 数字 | 是 | 系统自动生成的题目唯一编号 |
| 项目名称 | 文本 | 是 | 修改后的项目名称 |
| 项目内容及要求 | 长文本 | 是 | 修改后的项目要求 |
| 技术路线 | 长文本 | 是 | 修改后的技术路线 |
| 所需学生人数 | 数字 | 是 | 修改后的学生人数要求 |
| 修改原因 | 长文本 | 否 | 特殊情况下修改时填写 |

#### 6.2 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 修改成功提示 | 题目信息修改成功 |
| 修改失败提示 | 无权限或题目状态不允许修改 |
| 最新题目信息 | 返回修改后的题目详情 |

#### 6.2 业务规则
1. 教师只能修改本人创建的题目。
1. 草稿状态的题目可以自由修改。
1. 被退回修改的题目可以根据审核意见修改后再次提交。
1. 已审核通过但未开放的题目，如修改核心字段，应重新提交审核。
1. 已有学生选择的题目，不允许教师修改项目名称、学生人数、项目要求等核心信息。
1. 管理员可以在特殊情况下维护题目信息，但应保留修改记录。

#### 6.2 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 修改他人题目 | 提示“无权限修改该题目” |
| 题目已开放选题 | 提示“题目已开放，不能随意修改核心信息” |
| 修改后必填项为空 | 提示用户补全必填项 |
| 修改数据保存失败 | 提示“保存失败，请重试” |

### 6.3 功能三：资料上传

#### 6.3 功能描述

教师可以为本人发布的题目上传相关资料，包括项目指导书、任务书、参考资料、数据文件、代码模板、开发说明等。学生在题目开放后可以查看和下载相关资料。

#### 6.3 使用角色

教师/学生/管理员

#### 6.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 题目编号 | 数字 | 是 | 资料所属题目 |
| 文件名称 | 文本 | 是 | 上传文件名称 |
| 文件类型 | 文本 | 是 | 如 docx、pdf、zip、pptx 等 |
| 文件大小 | 数字 | 是 | 系统自动记录 |
| 上传人 | 文本 | 是 | 系统自动获取当前用户 |
| 上传时间 | 日期时间 | 是 | 系统自动生成 |
| 文件说明 | 文本 | 否 | 对文件内容进行说明 |

#### 6.3 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 上传成功提示 | 文件上传成功 |
| 文件列表 | 显示该题目下已上传资料 |
| 下载链接 | 学生或教师可以下载资料 |
| 错误提示 | 文件格式、大小或权限不符合要求时提示原因 |

#### 6.3 业务规则
1. 教师只能为本人创建的题目上传资料。
1. 管理员可以管理所有题目的资料。
1. 学生只能查看和下载已开放题目的相关资料。
1. 系统应限制上传文件格式，防止上传无关或危险文件。
1. 文件大小应设置上限，例如单个文件不超过 50MB。
1. 同一题目下允许上传多个资料文件。
1. 删除资料时应进行二次确认，避免误删。

#### 6.3 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 文件为空 | 提示“请选择上传文件” |
| 文件格式不支持 | 提示“文件格式不符合要求” |
| 文件超过大小限制 | 提示“文件过大，请重新选择” |
| 上传过程中断 | 提示“上传失败，请重新上传” |
| 学生尝试上传资料 | 提示“无上传权限” |

### 6.4 功能四：题目审核

#### 6.4 功能描述

管理员可以对教师提交的题目进行审核。审核时需要查看题目的基本信息、项目内容、技术路线、项目难度、学生人数要求和附件资料等内容，并给出审核结果和审核意见。

#### 6.4 使用角色

管理员

#### 6.4 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 题目编号 | 数字 | 是 | 被审核的题目编号 |
| 审核人 | 文本 | 是 | 当前管理员 |
| 审核结果 | 文本/下拉选择 | 是 | 审核通过、退回修改、不予通过 |
| 审核意见 | 长文本 | 是 | 对题目的审核说明 |
| 审核时间 | 日期时间 | 是 | 系统自动生成 |

#### 6.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 审核成功提示 | 审核结果保存成功 |
| 审核状态更新 | 题目状态变为审核通过、退回修改或不予通过 |
| 审核意见 | 教师可以查看审核意见 |
| 审核记录 | 系统保存历史审核信息 |

#### 6.4 业务规则
1. 只有管理员可以审核题目。
1. 审核通过后，题目进入题库。
1. 退回修改时，必须填写审核意见。
1. 不予通过时，必须说明原因。
1. 审核记录应长期保留，教师不能删除。
1. 题目审核通过后，是否开放给学生选择由管理员控制。
1. 审核通过的题目如再次修改核心信息，应重新提交审核。

#### 6.4 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 审核意见为空 | 提示“请填写审核意见” |
| 教师尝试审核题目 | 提示“无审核权限” |
| 学生尝试审核题目 | 提示“无审核权限” |
| 重复审核同一题目 | 提示“该题目已审核，请勿重复操作” |
| 题目信息不存在 | 提示“题目信息不存在” |

### 6.5 功能五：题库管理

#### 6.5 功能描述

题库管理用于对系统中的所有题目进行统一维护，包括题目查询、题目状态管理、开放选题、关闭选题、归档历史题目等。

#### 6.5 使用角色

管理员/教师/学生

#### 6.5 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 题目编号 | 数字 | 是 | 题目的唯一编号 |
| 题目状态 | 文本/数字 | 是 | 草稿、待审核、审核通过、退回修改、不予通过等 |
| 开放状态 | 文本/数字 | 是 | 未开放、已开放、已关闭 |
| 归档状态 | 文本/数字 | 否 | 是否归档 |
| 操作人 | 文本 | 是 | 当前操作用户 |
| 操作时间 | 日期时间 | 是 | 系统自动生成 |

#### 6.5 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 题目列表 | 显示系统中的题目信息 |
| 状态更新提示 | 开放、关闭、归档等操作成功提示 |
| 查询结果 | 返回符合条件的题目 |
| 错误提示 | 操作失败时提示原因 |

#### 6.5 业务规则
1. 只有审核通过的题目才能开放给学生选择。
1. 已开放题目可以被学生查看。
1. 达到人数上限的题目应自动变为“已满员”。
1. 历史题目可以归档，但不应影响历史记录查询。
1. 已被学生选择的题目不允许直接删除。
1. 管理员可以关闭题目，关闭后学生不能继续选择。
1. 学生只能查看已审核通过并已开放的题目。

#### 6.5 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 未审核题目被开放 | 提示“题目未审核通过，不能开放” |
| 删除已被选择的题目 | 提示“该题目已有学生选择，不能删除” |
| 学生查看未开放题目 | 系统不显示该题目 |
| 题目人数已满 | 提示“该题目已满员，不能继续选择” |

## 7. 模块业务流程
1. 教师登录系统。
1. 教师进入出题管理页面。
1. 教师填写项目题目信息。
1. 教师上传项目指导书、参考资料等系统资料。
1. 教师保存草稿或提交审核。
1. 系统对题目信息进行完整性校验。
1. 校验通过后，题目状态变为“待审核”。
1. 管理员查看待审核题目。
1. 管理员根据项目内容、技术路线、难度、人数要求等进行审核。
1. 如果审核通过，题目进入题库。
1. 如果审核不通过或需要修改，题目退回教师。
1. 教师根据审核意见修改后重新提交。
1. 管理员将审核通过的题目设置为“开放选题”。
1. 学生可以查看已开放的题目。
1. 题目达到人数上限后，系统自动设置为“已满员”。

## 15. 备注
1. 本模块是否需要支持 Excel 批量导入题目，需要由小组统一决定。
1. 删除题目建议采用逻辑删除，不建议直接物理删除。
1. 文件上传大小需要统一限制，建议单个文件不超过 50MB。
1. 题目审核通过后是否允许教师修改，需要与选题管理模块统一规则。
1. 已经被学生选择的题目不应允许随意删除或修改核心内容。
1. 题目状态编码需要与数据库设计统一，例如草稿、待审核、审核通过、退回修改、不予通过等。
1. 题目是否开放给学生选择，应由管理员统一控制。
1. 后续如果时间充足，可以增加题目重复检测、题目导出 Excel、题目统计分析等扩展功能。

# 选题管理模块需求分析

## 1. 模块名称

选题管理模块

## 2. 模块负责人

| 项目 | 内容 |
| --- | --- |
| 姓名 | 高乐 |
| 负责内容 | 学生组队、课题浏览与查询、选题申请、教师审核、过程文档提交、开发日志管理等功能的需求分析、数据库设计、页面设计与代码实现。 |

## 3. 模块目标

本模块主要用于管理综合实训项目中学生从组队、浏览课题、提交选题申请，到教师审核、过程文档提交及开发日志记录的全过程。

通过该模块，学生能够以团队形式完成项目选题，并在项目实施过程中持续提交相关文档和开发记录；指导教师能够及时了解团队选题情况、项目进展及学生参与情况，为后续过程考核和成绩评定提供依据。

本模块实现学生组队、选题完成、提交过程文档及开发日志管理等业务功能，是综合实训项目从课题发布到过程管理的重要衔接环节。

## 4. 使用角色

| 角色 | 权限说明 |
| --- | --- |
| 学生 | 创建或加入团队，浏览课题，提交过程文档，填写开发日志，查看本人团队的选题申请状态和教师反馈。 |
| 团队负责人 | 除普通学生权限外，还可维护团队成员、提交或撤销选题申请、查看团队整体文档及日志提交情况。 |
| 教师 | 查看本人课题的选题申请，对选题进行审核，查看团队过程文档和开发日志，并填写指导反馈。 |
| 管理员 | 维护选题开放时间、处理异常选题和团队信息，查看全部选题数据并进行系统管理。 |

## 6. 功能详细说明

### 6.1 功能一：学生组队管理

#### 6.1.1 功能描述

学生在选题前需要先完成组队。学生可以创建团队、申请加入团队或退出尚未完成选题的团队。创建团队的学生默认成为团队负责人，负责维护团队成员和代表团队提交选题申请。

#### 6.1.2 使用角色

学生、团队负责人、管理员。

#### 6.1.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 团队名称 | 文本 | 是 | 团队的名称，要求具有唯一性。 |
| 团队简介 | 文本 | 否 | 简要说明团队研究方向或项目计划。 |
| 团队负责人 | 文本/编号 | 是 | 创建团队的学生，系统自动生成。 |
| 团队成员 | 文本/编号 | 否 | 团队成员信息。 |
| 成员分工 | 文本 | 否 | 记录每位成员的初步职责。 |

#### 6.1.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 创建成功提示 | 团队创建完成，当前学生成为负责人。 |
| 团队信息列表 | 显示团队名称、成员人数、负责人、状态等信息。 |
| 加入申请结果 | 显示加入团队是否成功或等待负责人审核。 |
| 错误提示 | 提示团队名称重复、学生已加入其他团队等问题。 |

#### 6.1.5 业务规则

每名学生在同一实训周期内只能加入一个团队。

团队负责人由创建团队的学生担任。

团队名称不能重复。

团队人数不得超过课题规定的最大人数。

选题审核通过后，团队成员不能随意退出或加入。

团队负责人可以维护成员分工信息。

#### 6.1.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 团队名称为空 | 提示“请输入团队名称”。 |
| 团队名称重复 | 提示“该团队名称已存在”。 |
| 学生已加入其他团队 | 提示“当前学生已加入其他团队，不能重复加入”。 |
| 团队人数已满 | 提示“该团队成员数量已达到上限”。 |
| 已完成选题后修改成员 | 提示“当前团队已完成选题，请联系教师或管理员处理”。 |

### 6.2 功能二：课题浏览与查询

#### 6.2.1 功能描述

学生可查看指导教师发布并审核通过的课题信息，根据课题方向、指导教师、难度和选题状态等条件筛选适合本团队的综合实训课题。

#### 6.2.2 使用角色

学生、教师、管理员。

#### 6.2.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 课题关键词 | 文本 | 否 | 按课题名称进行模糊查询。 |
| 指导教师 | 文本/编号 | 否 | 按教师姓名或工号查询。 |
| 课题方向 | 文本 | 否 | 如Java Web、Android、数据库设计等。 |
| 项目难度 | 文本 | 否 | 如A、B、C等难度等级。 |
| 课题状态 | 文本 | 否 | 可选、审核中、已选满、已关闭等。 |

#### 6.2.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 课题列表 | 显示课题名称、指导教师、人数要求、难度及状态。 |
| 课题详情 | 显示项目简介、功能要求、技术要求、参考资料等。 |
| 无数据提示 | 没有符合条件的课题时进行提示。 |

#### 6.2.5 业务规则

学生只能查看已审核通过且处于“可选”状态的课题。

已选满或已关闭的课题不能提交选题申请。

课题详情中应展示项目人数要求和指导教师信息。

学生可按关键词、教师、难度和状态进行组合查询。

#### 6.2.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 查询条件为空 | 默认展示全部可选课题。 |
| 查询无结果 | 提示“未查询到符合条件的课题”。 |
| 课题已关闭 | 提示“该课题当前不可选择”。 |
| 课题人数已满 | 提示“该课题已选满，请选择其他课题”。 |

### 6.3 功能三：选题申请管理

#### 6.3.1 功能描述

团队负责人代表本团队选择课题，并填写选题说明、成员分工等信息后提交选题申请。学生可查看申请状态和教师审核意见。

#### 6.3.2 使用角色

团队负责人、学生。

#### 6.3.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 课题编号 | 数字/编号 | 是 | 所选择的课题。 |
| 团队编号 | 数字/编号 | 是 | 当前申请团队。 |
| 选题说明 | 文本 | 是 | 说明选择该课题的理由和实施思路。 |
| 团队成员 | 文本/编号 | 是 | 团队成员名单，由系统自动获取。 |
| 成员分工 | 文本 | 是 | 填写每位成员的初步职责。 |
| 申请时间 | 日期时间 | 是 | 系统自动生成。 |

#### 6.3.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 提交成功提示 | 选题申请已提交，等待教师审核。 |
| 申请记录 | 显示课题名称、申请时间、审核状态和审核意见。 |
| 审核结果 | 显示“待审核”“审核通过”或“审核驳回”。 |
| 错误提示 | 提示重复申请、团队未组建完成等问题。 |

#### 6.3.5 业务规则

只有团队负责人可以提交选题申请。

团队必须至少包含规定人数后才可提交申请。

同一团队同一时间只能提交一条待审核申请。

已审核通过的团队不能再次申请其他课题。

选题被驳回后，团队可修改申请信息并重新提交。

系统需记录选题申请的提交时间、审核时间和审核意见。

#### 6.3.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 非团队负责人提交申请 | 提示“仅团队负责人可以提交选题申请”。 |
| 未完成组队 | 提示“请先完成团队组建”。 |
| 已存在待审核申请 | 提示“当前团队已有待审核选题申请”。 |
| 已审核通过其他课题 | 提示“当前团队已完成选题，不能重复申请”。 |
| 课题已被选满 | 提示“该课题已无剩余名额”。 |

### 6.4 功能四：选题审核管理

#### 6.4.1 功能描述

教师查看申请其本人课题的团队信息，并根据团队人数、成员分工和选题说明进行审核。教师可通过或驳回申请，并填写审核意见。

#### 6.4.2 使用角色

教师、管理员。

#### 6.4.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 申请编号 | 数字/编号 | 是 | 待审核的选题申请记录。 |
| 审核结果 | 文本 | 是 | 通过或驳回。 |
| 审核意见 | 文本 | 驳回时必填 | 填写审核意见或修改建议。 |
| 审核时间 | 日期时间 | 是 | 系统自动生成。 |
| 审核教师 | 文本/编号 | 是 | 系统根据登录用户自动获取。 |

#### 6.4.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 审核成功提示 | 提示审核操作已完成。 |
| 审核结果 | 向学生显示审核通过或驳回状态。 |
| 课题状态更新 | 审核通过后更新课题已选人数和可选状态。 |
| 错误提示 | 提示无权限、申请状态异常等问题。 |

#### 6.4.5 业务规则

教师只能审核本人发布课题的申请。

审核通过后，系统自动将团队与课题绑定。

审核驳回时，教师必须填写驳回原因。

课题达到人数上限后，状态自动调整为“已选满”。

已审核完成的申请不能重复审核。

管理员可在特殊情况下对审核结果进行维护。

#### 6.4.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 教师审核非本人课题 | 提示“无审核权限”。 |
| 审核意见为空 | 提示“驳回申请时必须填写审核意见”。 |
| 申请已被处理 | 提示“该申请已审核，不能重复操作”。 |
| 课题人数已满 | 提示“该课题人数已满，无法继续审核通过”。 |

### 6.5 功能五：过程文档管理

| 文件附件 | 文件 | 是 | Word、PDF、Excel、压缩包等。 |
| 文档说明 | 文本 | 否 | 说明文档内容或本次修改情况。 |

#### 6.5.4 输出结果

| 输出内容 | 说明 |
| --- | --- |

## 7. 模块业务流程

学生登录系统后，进入选题管理模块。

学生创建团队或申请加入已有团队。

团队负责人浏览可选课题，查看课题详细要求。

团队负责人选择课题，填写选题说明和成员分工后提交申请。

指导教师查看选题申请并进行审核。

审核通过后，系统将课题分配给对应团队；审核驳回后，团队根据意见修改并重新提交。

项目实施过程中，团队成员上传过程文档并定期填写开发日志。

指导教师查看团队文档和日志，了解项目进展并进行反馈。

管理员对选题时间、异常团队和异常数据进行统一维护。

## 6. 功能详细说明

### 6.1 功能一：在线签到

#### 6.1.1 功能描述

学生在教师发布的签到任务有效期内，通过系统进行在线签到。系统自动记录签到时间、签到状态（正常/迟到/缺勤），并关联到对应的签到任务和学生信息。

#### 6.1.2 使用角色

学生

#### 6.1.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 签到任务ID | 数字 | 是 | 系统自动获取当前有效的签到任务 |
| 学生ID | 数字 | 是 | 系统自动获取当前登录学生信息 |
| 签到时间 | 日期时间 | 是 | 系统自动记录当前时间 |
| 签到备注 | 文本 | 否 | 学生可填写签到相关说明 |

#### 6.1.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 签到成功提示 | 显示签到成功，并提示签到状态（正常/迟到） |
| 签到失败提示 | 显示失败原因，如不在签到时间内、已签到过等 |
| 签到记录 | 返回本次签到的详细信息 |

#### 6.1.5 业务规则

学生只能在签到任务规定的时间范围内进行签到。

在签到开始时间后15分钟内签到为正常签到，超过15分钟为迟到签到。

每个学生对同一个签到任务只能签到一次，不允许重复签到。

签到结束后未签到的学生自动标记为缺勤。

学生只能查看和参与本人所属班级/队伍的签到任务。

#### 6.1.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |

| 已完成签到 | 提示"您已完成本次签到，请勿重复签到" |
| 无有效签到任务 | 提示"当前没有进行中的签到任务" |

### 6.3 功能三：补签申请与审核

#### 6.3.1 功能描述

学生因请假、网络异常、设备故障等特殊原因未能正常签到时，可以提交补签申请。教师查看申请原因和相关说明后进行审核，审核通过后系统将对应考勤记录标记为补签；审核驳回时保留驳回原因，供学生查看。

#### 6.3.2 使用角色

学生、教师

#### 6.3.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 签到任务ID | 数字 | 是 | 需要补签的签到任务 |
| 学生ID | 数字 | 是 | 系统自动获取当前登录学生 |
| 补签原因 | 长文本 | 是 | 说明未正常签到原因 |
| 证明材料路径 | 文本 | 否 | 可选附件或图片路径 |
| 审核结果 | 下拉选择 | 教师审核时必填 | 通过/驳回 |
| 审核意见 | 长文本 | 教师审核时必填 | 教师审核说明 |

#### 6.3.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 申请提交成功提示 | 显示补签申请已提交，等待审核 |
| 审核结果提示 | 显示补签申请通过或驳回 |
| 补签申请列表 | 返回学生本人或教师可审核范围内的补签申请 |
| 考勤记录更新 | 审核通过后，签到状态更新为补签 |

#### 6.3.5 业务规则

学生只能为本人缺勤或异常签到记录提交补签申请。

同一个签到任务同一学生只能存在一条待审核补签申请。

补签申请通过后，对应考勤记录的 `sign_status` 更新为补签。

教师只能审核本人负责班级或队伍范围内的补签申请。

补签申请和审核记录应长期保留，不能物理删除。

#### 6.3.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 重复提交补签申请 | 提示"该签到任务已有待审核补签申请" |
| 非本人考勤记录 | 提示"只能为本人考勤记录申请补签" |
| 无权限审核 | 提示"无补签审核权限" |
| 审核意见为空 | 提示"请填写审核意见" |

## 7. 模块业务流程

教师登录系统，进入考勤管理模块。

教师发布签到任务，设置签到时间、适用范围等参数。

系统自动通知相关学生有新的签到任务。

学生在规定时间内登录系统进行在线签到。

系统记录签到时间，自动判断签到状态（正常/迟到/缺勤）。

学生如因特殊原因未正常签到，可提交补签申请。

教师查看补签申请，审核通过或驳回并填写审核意见。

系统自动统计考勤数据，生成考勤报表。

教师和学生可以随时查询考勤记录和补签处理情况。

教师可导出考勤报表，用于成绩评定和存档。

#### 6.3.2 使用角色

学生、教师、管理员

#### 6.3.3 输入数据

| 公告类型 | 下拉选择 | 是 | 实训通知/截止提醒/系统通知 |
| 置顶标识 | 数字 | 是 | 0不置顶 1置顶 |
| 附件文件 | 文件 | 否 | Word、PDF实训模板等 |

#### 6.3.4 输出结果

### 6.4 功能四：系统数据备份管理

#### 6.4.1 功能描述

管理员手动一键备份全库数据，支持设置定时自动备份；查看所有备份记录，下载备份文件、删除过期备份；支持数据库恢复操作，保障实训数据不丢失。

#### 6.4.2 使用角色

管理员

#### 6.4.3 输入数据

## 7. 模块业务流程

系统管理员登录，进入信息管理主页面；

管理员按需维护学生、教师账号，批量导入导出人员数据；

管理员/教师进入公告页面，填写公告内容、上传附件并发布；

学生、教师登录首页浏览全部公告，下载配套实训文档；

管理员进入数据备份页面，手动执行数据库备份或配置自动备份周期；

系统生成备份文件并保存备份记录，可随时下载、删除过期备份；

出现数据丢失时，管理员选择历史备份文件，确认后执行数据库恢复；

所有新增、修改、删除、备份操作自动写入操作日志，管理员可追溯查看。

## 6. 功能详细说明

### 6.1 功能一：阶段任务发布

#### 6.1.1 功能描述

教师根据实训计划，分阶段发布任务要求。每个阶段任务包含任务名称、任务描述、交付物要求、截止时间、评分标准、权重占比等信息。发布后学生可以查看并按要求完成任务。

#### 6.1.2 使用角色

教师

#### 6.1.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 阶段名称 | 文本 | 是 | 如"需求分析阶段"、"设计阶段"等 |
| 阶段描述 | 富文本 | 是 | 阶段任务的详细要求和说明 |
| 开始时间 | 日期时间 | 是 | 阶段任务开始时间 |
| 截止时间 | 日期时间 | 是 | 阶段任务提交截止时间 |
| 交付物要求 | 文本 | 是 | 需要提交的成果物清单及要求 |
| 评分标准 | 富文本 | 是 | 本阶段的评分细则和标准 |
| 权重占比 | 数字 | 是 | 本阶段成绩在总成绩中的占比（百分比） |
| 适用范围 | 多选 | 是 | 适用的班级或队伍 |

#### 6.1.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 发布成功提示 | 显示阶段任务发布成功 |
| 阶段任务列表 | 返回所有阶段任务及状态 |
| 错误提示 | 显示发布失败的原因 |

#### 6.1.5 业务规则

截止时间必须晚于开始时间。

所有阶段的权重占比之和应为100%，系统自动校验。

教师只能为自己所指导的班级或队伍发布阶段任务。

阶段任务发布后，在开始时间前可以修改，开始后只能修改截止时间和描述。

阶段任务发布后，系统自动通知相关学生。

支持阶段任务的模板化管理，可复用历史阶段任务模板。

#### 6.1.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 时间设置不合理 | 提示"截止时间必须晚于开始时间" |
| 权重总和不为100% | 提示"所有阶段权重之和必须为100%，当前为X%" |
| 未选择适用范围 | 提示"请至少选择一个班级或队伍" |
| 阶段名称重复 | 提示"该阶段名称已存在，请使用其他名称" |

| 完成度得分 | 数字 | 是 | 任务完成情况评分（0-100） |

| 技术难度得分 | 数字 | 否 | 技术实现难度评分（0-100） |
| 评价意见 | 富文本 | 是 | 详细的评价意见和改进建议 |
| 是否通过 | 单选 | 是 | 通过/需修改/不通过 |

#### 6.2.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 评价成功提示 | 显示评价提交成功 |
| 阶段成绩 | 自动计算并返回本阶段的综合得分 |
| 评价详情 | 返回完整的评价信息 |

#### 6.2.5 业务规则

各维度得分按预设权重计算综合得分，权重可由管理员配置。

评价提交后，学生可查看评价结果和教师评语。

评价结果为"需修改"的，学生可重新提交，教师重新评价。

教师只能评价自己所指导的学生或队伍。

逾期提交的成果按规定扣分，扣分规则可配置。

评价完成后，系统自动更新阶段成绩和总成绩。

#### 6.2.6 异常情况

| --- | --- |

| 重复评价 | 提示"该队伍本阶段已评价，是否修改原有评价" |
| 无权限评价 | 提示"您无权评价该队伍/学生" |

### 6.3 功能三：成员贡献度记录

#### 6.3.1 功能描述

记录项目队伍中各成员的贡献度，支持学生互评和教师评价相结合的方式。贡献度作为个人成绩调整的重要依据，确保成绩评定的公平性。

#### 6.3.2 使用角色

学生、教师

#### 6.3.3 输入数据

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 队伍ID | 数字 | 是 | 所属队伍 |
| 被评价成员ID | 数字 | 是 | 被评价的学生 |
| 评价人ID | 数字 | 是 | 进行评价的学生或教师 |
| 贡献度评分 | 数字 | 是 | 贡献度评分（0-100） |
| 工作量占比 | 数字 | 是 | 该成员承担的工作量比例（%） |
| 工作内容描述 | 文本 | 是 | 该成员具体完成的工作内容 |
| 评价说明 | 文本 | 否 | 评价的补充说明 |
| 评价类型 | 单选 | 是 | 互评/教师评价/自评 |

#### 6.3.4 输出结果

| 输出内容 | 说明 |
| --- | --- |
| 提交成功提示 | 显示贡献度评价提交成功 |
| 贡献度汇总 | 返回各成员的综合贡献度排名 |
| 评价详情列表 | 返回所有评价的详细信息 |

#### 6.3.5 业务规则

队伍内所有成员的工作量占比之和应为100%。

学生互评占贡献度最终得分的60%，教师评价占40%（权重可配置）。

每个学生只能对同队伍的其他成员进行一次评价。

贡献度评价在每个阶段结束后开放，在规定时间内完成。

教师可以查看所有学生的互评结果，并进行最终的教师评价。

贡献度结果将作为个人成绩调整系数，影响最终个人成绩。

#### 6.3.6 异常情况

| 异常情况 | 系统处理 |
| --- | --- |
| 工作量占比总和不为100% | 提示"所有成员工作量占比之和必须为100%，当前为X%" |
| 评分超出范围 | 提示"贡献度评分必须在0-100之间" |
| 重复评价 | 提示"您已对该成员进行过评价，是否修改" |
| 评价自己 | 提示"不能对自己进行互评（自评除外）" |
| 非本队伍成员 | 提示"您只能评价本队伍的成员" |

## 7. 模块业务流程

教师登录系统，进入成绩评估与过程分析模块。

教师根据实训计划发布各阶段任务，设置任务要求、截止时间和评分标准。

系统自动通知相关学生有新的阶段任务。

学生查看阶段任务要求，并在选题管理模块提交对应过程文档或开发日志。

教师查看各队伍在选题管理模块中的过程文档和开发日志，对阶段成果进行评价和打分。

系统自动记录阶段成绩，并更新项目进度。

阶段结束后，学生进行成员贡献度互评。

教师查看互评结果，进行教师评价，确定最终贡献度。

教师通过进度看板实时跟踪各队伍的项目进度，及时发现问题。

所有阶段完成后，系统自动汇总各阶段成绩、考勤成绩、答辩成绩等。

教师进行最终评分，结合贡献度调整个人成绩。

系统计算最终成绩，生成成绩报表。

教师和管理员可导出成绩数据，用于存档和上报。

学生可随时查看个人成绩详情和各阶段评价。

## 15. 备注

成绩评定的具体权重分配（文档15%、考勤15%、系统实现50%、答辩20%）是否需要支持灵活配置？目前按指导书的比例设计。

贡献度评价的具体权重分配（互评60%、教师评价40%）是否需要调整？是否需要加入自评？

成绩等级的划分标准是什么？如多少分以上为优秀，是否需要支持自定义等级和分数线？

是否需要支持成绩申诉功能？学生对成绩有异议时可以提交申诉。

阶段任务是否需要支持子任务拆分？即一个大阶段下包含多个小任务。

进度看板是否需要支持甘特图形式的可视化展示？

是否需要支持批量导入成绩？如教师有Excel成绩表可以批量导入系统。

答辩成绩如何录入？是否需要支持多位评委打分后取平均分？

是否需要支持成绩排名功能？包括班级排名和队伍内排名。

迟交扣分的具体规则是什么？如每天扣多少分，扣分上限是多少？

是否需要支持成绩锁定功能？成绩确认后锁定，不允许随意修改。

成员贡献度评价是每个阶段都做，还是只在项目结束时做一次？
