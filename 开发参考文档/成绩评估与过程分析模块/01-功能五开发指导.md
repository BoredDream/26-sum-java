# 功能五：成绩评价与过程监控管理开发指导

## 1. 模块范围

本模块对应 `backend/src/main/java/com/training/system/score`，用于实现综合实训的阶段任务、过程评价、进度跟踪、贡献度、成绩录入、成绩确认、成绩查询和导出。

模块只读取选题、团队、过程文档、考勤等过程数据，不直接修改其他模块原始数据。

## 2. 使用数据表

| 表名 | 用途 |
| --- | --- |
| `stage_task` | 阶段任务发布与查询 |
| `stage_evaluation` | 阶段成果评价 |
| `contribution` | 成员贡献度记录 |
| `score` | 团队成绩主表 |
| `student_score` | 个人成绩表 |
| `team_info` | 读取团队信息 |
| `topic_selection` | 读取团队选题结果 |
| `project_topic` | 读取题目信息 |
| `student`、`teacher` | 读取学生、教师基础信息 |

## 3. 接口清单

| 功能 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| 发布阶段任务 | POST | `/api/score/stage-task` | 教师/管理员发布阶段任务 |
| 查询阶段任务 | GET | `/api/score/stage-task/page` | 分页查询阶段任务 |
| 阶段成果评价 | POST | `/api/score/stage-evaluation` | 教师/管理员评价团队阶段成果 |
| 项目进度跟踪 | GET | `/api/score/progress` | 查看团队选题、阶段评价数量和阶段均分 |
| 成员贡献度记录 | POST | `/api/score/contribution` | 学生互评、教师评价、自评 |
| 保存团队成绩 | POST | `/api/score` | 保存团队分项成绩并生成个人成绩 |
| 查询团队成绩 | GET | `/api/score/page` | 分页查询团队成绩 |
| 查询个人成绩 | GET | `/api/score/student/{studentId}` | 查询学生个人成绩 |
| 教师确认成绩 | POST | `/api/score/{scoreId}/confirm` | 将成绩状态改为已确认 |
| 成绩导出 | GET | `/api/score/export` | 导出 CSV 成绩表 |

## 4. 权限规则

1. 阶段任务发布、阶段评价、团队成绩保存、成绩确认、成绩导出仅教师和管理员可操作。
2. 学生可以查询自己的个人成绩，可以提交贡献度评价。
3. 教师只能管理本人评分数据；管理员可查看全部。
4. 已锁定成绩不允许再次修改。
5. AI 只提供建议，最终成绩以教师确认结果为准。

## 5. 计算规则

团队总成绩：

```text
total_score = doc_score + attendance_score + system_score + defense_score
```

默认满分约束：

| 项目 | 满分 |
| --- | --- |
| 文档成绩 | 15 |
| 考勤成绩 | 15 |
| 系统实现与测试 | 50 |
| 答辩成绩 | 20 |

个人成绩：

```text
personal_score = team_total_score * contribution_factor
```

成绩等级：

| 分数 | 等级 |
| --- | --- |
| >= 90 | 优秀 |
| >= 80 | 良好 |
| >= 70 | 中等 |
| >= 60 | 及格 |
| < 60 | 不及格 |

## 6. 实现顺序

1. 创建 Entity、DTO、VO。
2. 创建 Mapper 接口和 XML。
3. 编写 `ScoreService` 和 `ScoreServiceImpl`。
4. 编写 `ScoreController`。
5. 运行 `mvn test`，确认项目可编译。
