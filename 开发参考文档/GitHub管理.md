# GitHub 管理规范

本文档用于统一《基于 Java 的综合实训选题系统》的 GitHub 仓库协作方式。所有成员必须基于同一个 GitHub 仓库、同一个项目骨架、同一套数据库设计 V1 开发，禁止每个人单独新建项目后再合并。

## 1. 仓库信息

| 项目 | 约定 |
|---|---|
| 远程仓库 | `git@github.com:BoredDream/26-sum-java.git` |
| 默认稳定分支 | `main` |
| 集成开发分支 | `dev` |
| 功能开发分支 | `feature/模块名` |
| 合并方式 | 功能分支提交 Pull Request 到 `dev` |
| 合并负责人 | 项目负责人或技术负责人统一审核并合并 |

## 2. 分支规则

| 分支 | 用途 | 规则 |
|---|---|---|
| `main` | 稳定版本 | 不允许直接提交，阶段性稳定后由负责人从 `dev` 合并 |
| `dev` | 集成开发分支 | 各模块通过 PR 合并到此分支，用于联调 |
| `feature/topic` | 出题管理模块 | 出题模块负责人开发使用 |
| `feature/selection` | 选题管理模块 | 选题模块负责人开发使用 |
| `feature/attendance` | 学生考勤管理模块 | 考勤模块负责人开发使用 |
| `feature/info` | 信息管理模块 | 信息模块负责人开发使用 |
| `feature/score` | 成绩评估与过程分析模块 | 成绩模块负责人开发使用 |

## 3. 开发流程

1. 所有成员先克隆同一个 GitHub 仓库。
2. 项目负责人创建并维护 `main`、`dev` 分支。
3. 每个成员从 `dev` 拉取自己的功能分支。
4. 每个成员只在自己负责的模块目录内开发。
5. 修改公共代码、公共配置、数据库 SQL 前，必须先通知项目负责人。
6. 功能完成后提交 Pull Request 到 `dev`。
7. PR 通过审核后，由项目负责人或技术负责人合并。
8. 每周至少进行一次 `dev` 分支联调。

## 4. 提交信息规范

提交信息格式：

```text
模块名: 本次修改内容
```

示例：

```text
出题管理: 完成题目新增接口
选题管理: 新增团队创建接口
考勤管理: 完成补签申请表设计
信息管理: 添加 user_account 表结构
成绩评估与过程分析: 添加 score 和 student_score 表说明
```

禁止使用以下提交信息：

```text
更新
修改
test
111
随便改一下
```

## 5. Pull Request 规范

PR 标题格式：

```text
[模块名] 本次合并内容
```

PR 描述必须包含：

| 项目 | 要求 |
|---|---|
| 完成功能 | 说明本次完成了哪些功能 |
| 修改文件 | 说明改动了哪些文件 |
| 数据库变更 | 如有表结构或字段变化，必须写明 |
| 接口变更 | 如有新增或修改接口，必须写明路径和参数 |
| 测试情况 | 说明本地是否能启动、接口是否测试通过 |
| 影响范围 | 是否影响其他模块或公共代码 |
| 未完成事项 | 如有遗留问题，必须列出 |

## 6. 公共文件修改规则

以下内容属于公共区域，不允许个人随意修改：

| 公共内容 | 修改要求 |
|---|---|
| `backend/src/main/java/.../common` | 修改前通知负责人 |
| `backend/src/main/java/.../config` | 修改前通知负责人 |
| `backend/src/main/resources/application-example.yml` | 修改前通知负责人 |
| `database/sql/schema-v1.sql` | 只能由负责人确认后合并 |
| `database/docs/data-dictionary.md` | 修改前说明影响范围 |
| `pom.xml` | 新增依赖前说明用途 |

## 7. 数据库变更规则

1. 各模块负责人可以提出表结构和字段建议。
2. 最终数据库 V1 由项目负责人统一确认。
3. 不允许各模块私自建表后直接开发。
4. 数据库 SQL 统一放在 `database/sql/`。
5. 字段变更必须同步更新数据字典。
6. PR 中必须说明数据库变更原因和影响模块。

## 8. 禁止提交内容

`.gitignore` 必须排除以下内容：

```text
target/
*.class
*.log
.idea/
*.iml
.vscode/
node_modules/
dist/
.env
application-local.yml
application-dev.yml
.DS_Store
Thumbs.db
```

禁止提交：

1. 本地数据库账号和密码。
2. API Key、GitHub Token、AI Key。
3. `target/`、`node_modules/`、日志文件。
4. 个人 IDE 配置。
5. 未经确认的数据库建表 SQL。

## 9. 合并前检查清单

| 检查项 | 是否必须 |
|---|---|
| 项目可以正常启动 | 是 |
| 没有编译错误 | 是 |
| 没有提交敏感信息 | 是 |
| 没有提交无关文件 | 是 |
| 接口返回格式统一 | 是 |
| 数据库变更已说明 | 是 |
| 公共代码变更已通知 | 是 |
| PR 描述完整 | 是 |

## 10. 当前启动建议

项目正式分模块编码前，应先完成：

1. 数据库总表结构 V1。
2. 后端 Spring Boot 最小项目骨架。
3. `dev` 与各 `feature/*` 分支创建。
4. 每个模块接口清单。
5. 每个模块依赖表说明。
