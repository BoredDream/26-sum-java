## 3. 正常流程测试

### TC-N-01：教师新增题目并保存草稿

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师填写完整题目信息后保存为草稿 |
| 前置条件 | 教师已登录，角色为 TEACHER |
| 输入 | `TopicCreateDTO`：topicName="在线商城系统", topicType="应用系统", difficulty="B", studentLimit=4, topicContent="开发一个完整的B2C商城...", technicalRoute="Spring Boot + Vue + MySQL", status=0（草稿） |
| 预期结果 | 返回 code=200，题目保存成功，status=0（草稿） |

### TC-N-02：教师提交题目审核

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师填写完整题目信息后直接提交审核 |
| 前置条件 | 教师已登录 |
| 输入 | `TopicCreateDTO`：所有必填项完整填写，status=1（提交审核） |
| 预期结果 | 返回 code=200，题目提交成功，status=1（待审核） |

### TC-N-03：教师将草稿提交审核

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师将已有的草稿题目提交审核 |
| 前置条件 | 存在一条草稿状态的题目，且属于当前教师 |
| 输入 | `POST /api/topic/{topicId}/submit` |
| 预期结果 | 返回 code=200，题目状态变为 1（待审核） |

### TC-N-04：管理员审核通过题目

| 项目 | 内容 |
|---|---|
| 测试内容 | 管理员对待审核题目进行审核通过 |
| 前置条件 | 管理员已登录，存在一条待审核状态的题目 |
| 输入 | `TopicReviewDTO`：reviewResult=1（通过）, reviewComment="题目符合要求，予以通过" |
| 预期结果 | 返回 code=200，题目 status=2（审核通过），生成审核记录 |

### TC-N-05：管理员开放题目

| 项目 | 内容 |
|---|---|
| 测试内容 | 管理员对审核通过的题目进行开放操作 |
| 前置条件 | 管理员已登录，存在一条审核通过且未开放的题目 |
| 输入 | `POST /api/topic/{topicId}/open` |
| 预期结果 | 返回 code=200，题目 open_status=1（已开放） |

### TC-N-06：教师上传题目资料

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师为本人题目上传项目指导书 |
| 前置条件 | 教师已登录，存在属于该教师的题目 |
| 输入 | MultipartFile: "项目指导书.pdf" (≤50MB), fileDesc="版本1.0" |
| 预期结果 | 返回 code=200，文件上传成功，topic_file 表新增一条记录 |

### TC-N-07：学生查看已开放题目列表

| 项目 | 内容 |
|---|---|
| 测试内容 | 学生查看所有已审核通过且已开放的题目 |
| 前置条件 | 学生已登录，存在已开放题目 |
| 输入 | `GET /api/topic/page`（无筛选条件） |
| 预期结果 | 返回 code=200，只包含 status=2 且 open_status=1 的题目 |

### TC-N-08：教师查看本人题目列表

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师查看自己创建的题目 |
| 前置条件 | 教师已登录，教师已创建若干题目 |
| 输入 | `GET /api/topic/page` |
| 预期结果 | 返回 code=200，只包含当前教师创建的题目 |

### TC-N-09：管理员查看全部题目

| 项目 | 内容 |
|---|---|
| 测试内容 | 管理员查看系统中所有题目 |
| 前置条件 | 管理员已登录，系统中存在各教师创建的题目 |
| 输入 | `GET /api/topic/page` + 可选筛选条件 |
| 预期结果 | 返回 code=200，包含所有未被删除的题目 |

### TC-N-10：查看题目详情含审核记录

| 项目 | 内容 |
|---|---|
| 测试内容 | 查看题目的完整信息和审核历史 |
| 前置条件 | 题目存在且至少有一条审核记录 |
| 输入 | `GET /api/topic/{topicId}` |
| 预期结果 | 返回 code=200，包含题目详情、资料列表、审核记录 |

## 4. 异常流程测试

### TC-E-01：必填项为空提交

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师提交审核时项目名称为空 |
| 前置条件 | 教师已登录 |
| 输入 | topicName=""（空字符串），其他必填项正常 |
| 预期结果 | 返回 code=400，message="项目名称不能为空" |

### TC-E-02：学生人数小于1

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师设置学生人数为0 |
| 前置条件 | 教师已登录 |
| 输入 | studentLimit=0 |
| 预期结果 | 返回 code=400，message="学生人数必须大于0" |

### TC-E-03：修改他人题目

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师尝试修改另一个教师的题目 |
| 前置条件 | 教师A已登录，存在教师B创建的题目 |
| 输入 | `PUT /api/topic/{teacherB的topicId}` |
| 预期结果 | 返回 code=403，message="无权限修改该题目" |

### TC-E-04：修改已开放题目

| 项目 | 内容 |
|---|---|
| 测试内容 | 教师尝试修改已开放选题的题目 |
| 前置条件 | 题目 status=2（审核通过），open_status=1（已开放） |
| 输入 | `PUT /api/topic/{topicId}` 修改核心字段 |
| 预期结果 | 返回 code=409，message="题目已开放，不能随意修改" |

### TC-E-05：审核意见为空

|---|---|

| 输入 | reviewResult=2（退回修改），reviewComment="" |
| 预期结果 | 返回 code=400，message="请填写审核意见" |

### TC-E-06：重复审核同一题目

| 输入 | `POST /api/topic/{topicId}/review` |

### TC-E-09：删除已被选择的题目

|---|---|

| 输入 | `DELETE /api/topic/{topicId}` |
