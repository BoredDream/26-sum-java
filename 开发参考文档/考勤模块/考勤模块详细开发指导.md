## 7. Service 层设计

Service 层负责业务规则判断，关键状态流转必须加事务（`@Transactional`）。

### 7.1 发布签到任务核心逻辑

```java
@Override
@Transactional(rollbackFor = Exception.class)
public Long createTask(AttendanceTaskCreateDTO dto, Long teacherId) {
    // 1. 校验时间
    if (!dto.getEndTime().isAfter(dto.getStartTime())) {
        throw new BusinessException(ResultCode.BAD_REQUEST, "结束时间必须晚于开始时间");
    }

    // 2. 校验范围
    List<Long> studentIds = resolveScopeStudents(dto.getScopeType(), dto.getScopeValue());
    if (studentIds.isEmpty() && dto.getScopeType() != 3) {
        throw new BusinessException(ResultCode.BAD_REQUEST, "适用范围内没有学生");
    }

    // 3. 保存任务
    AttendanceTask task = new AttendanceTask();
    BeanUtils.copyProperties(dto, task);
    task.setTeacherId(teacherId);
    task.setStatus(0);
    attendanceTaskMapper.insert(task);

    // 4. 预生成考勤记录（缺勤状态）
    for (Long studentId : studentIds) {
        attendanceRecordMapper.insertInitRecord(task.getTaskId(), studentId);
    }

    return task.getTaskId();
}
```

### 7.2 在线签到核心逻辑

```java
@Override
@Transactional(rollbackFor = Exception.class)
public AttendanceRecordVO sign(AttendanceSignDTO dto, Long studentId) {
    // 1. 查询任务
    AttendanceTask task = attendanceTaskMapper.selectById(dto.getTaskId());
    if (task == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "签到任务不存在");
    }

    // 2. 校验范围
    if (!isStudentInScope(task, studentId)) {
        throw new BusinessException(ResultCode.FORBIDDEN, "您不在本次签到范围内");
    }

    // 3. 校验时间
    LocalDateTime now = LocalDateTime.now();
    if (now.isBefore(task.getStartTime()) || now.isAfter(task.getEndTime())) {
        throw new BusinessException(ResultCode.BAD_REQUEST, "当前不在签到时间内");
    }

    // 4. 查询记录
    AttendanceRecord record = attendanceRecordMapper.selectByTaskAndStudent(dto.getTaskId(), studentId);
    if (record == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "考勤记录不存在");
    }
    if (record.getSignStatus() != null && record.getSignStatus() != 0) {
        throw new BusinessException(ResultCode.CONFLICT, "您已完成本次签到，请勿重复签到");
    }

    // 5. 判定状态
    int signStatus;
    if (!now.isAfter(task.getStartTime().plusMinutes(15))) {
        signStatus = 1; // 正常
    } else {
        signStatus = 2; // 迟到
    }

    // 6. 更新记录
    record.setSignTime(now);
    record.setSignStatus(signStatus);
    record.setRemark(dto.getRemark());
    attendanceRecordMapper.update(record);

    return convertToVO(record);
}
```

### 7.3 补签申请核心逻辑

```java
@Override
@Transactional(rollbackFor = Exception.class)
public Long applyMakeup(MakeupApplyDTO dto, Long studentId) {
    // 1. 查询考勤记录
    AttendanceRecord record = attendanceRecordMapper.selectByTaskAndStudent(dto.getTaskId(), studentId);
    if (record == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "考勤记录不存在");
    }
    if (record.getSignStatus() == 1) {
        throw new BusinessException(ResultCode.BAD_REQUEST, "当前考勤记录无需补签");
    }

    // 2. 检查是否有待审核申请
    MakeupSignApply pending = makeupSignApplyMapper
            .selectPendingByTaskAndStudent(dto.getTaskId(), studentId);
    if (pending != null) {
        throw new BusinessException(ResultCode.CONFLICT, "该签到任务已有待审核补签申请");
    }

    // 3. 保存申请
    MakeupSignApply apply = new MakeupSignApply();
    apply.setTaskId(dto.getTaskId());
    apply.setRecordId(record.getRecordId());
    apply.setStudentId(studentId);
    apply.setApplyReason(dto.getApplyReason());
    apply.setProofFilePath(dto.getProofFilePath());
    makeupSignApplyMapper.insert(apply);

    return apply.getApplyId();
}
```

### 7.4 补签审核核心逻辑

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void reviewMakeup(Long applyId, MakeupReviewDTO dto, Long teacherId, String role) {
    // 1. 查询申请
    MakeupSignApply apply = makeupSignApplyMapper.selectById(applyId);
    if (apply == null) {
        throw new BusinessException(ResultCode.NOT_FOUND, "补签申请不存在");
    }
    if (apply.getAuditStatus() != 0) {
        throw new BusinessException(ResultCode.CONFLICT, "该申请已审核，不能重复操作");
    }

    // 2. 校验权限
    AttendanceTask task = attendanceTaskMapper.selectById(apply.getTaskId());
    if (!"ADMIN".equals(role) && !task.getTeacherId().equals(teacherId)) {
        throw new BusinessException(ResultCode.FORBIDDEN, "无补签审核权限");
    }

    // 3. 驳回必须填写意见
    if (dto.getAuditStatus() == 2 && (dto.getAuditComment() == null || dto.getAuditComment().isBlank())) {
        throw new BusinessException(ResultCode.BAD_REQUEST, "请填写审核意见");
    }

    // 4. 更新申请
    apply.setAuditStatus(dto.getAuditStatus());
    apply.setAuditTeacherId(teacherId);
    apply.setAuditComment(dto.getAuditComment());
    makeupSignApplyMapper.updateAudit(apply);

    // 5. 通过则更新考勤记录
    if (dto.getAuditStatus() == 1) {
        AttendanceRecord record = attendanceRecordMapper.selectById(apply.getRecordId());
        record.setSignStatus(3); // 补签
        record.setIsMakeup(1);
        attendanceRecordMapper.update(record);
    }
}
```

private Long getCurrentUserId(HttpSession session) {

在 `AttendanceTaskServiceImpl` 中提供统一方法，根据 `scope_type` 和 `scope_value` 解析学生ID列表：

```java
private List<Long> resolveScopeStudents(Integer scopeType, String scopeValue) {
    switch (scopeType) {
        case 1: // 班级
            return studentMapper.selectIdsByClassName(scopeValue);
        case 2: // 团队
            return teamMemberMapper.selectStudentIdsByTeamId(Long.valueOf(scopeValue));
        case 3: // 全部
            return studentMapper.selectAllIds();
        default:
            throw new BusinessException(ResultCode.BAD_REQUEST, "适用范围类型不正确");
    }
}
```

> 需要调用 `com.training.system.info.mapper.StudentMapper` 和 `com.training.system.selection.mapper.TeamMemberMapper`。

### 9.2 签到状态判定

| 场景 | 状态 |
| --- | --- |
| 当前时间 ≤ start_time + 15min | 正常（1） |
| start_time + 15min < 当前时间 ≤ end_time | 迟到（2） |
| 当前时间 > end_time | 不允许签到，视为缺勤（0） |

使用 `LocalDateTime` 比较：

```java
LocalDateTime deadline = task.getStartTime().plusMinutes(15);
int signStatus = now.isAfter(deadline) ? 2 : 1;
```

### 9.3 任务状态维护

任务状态建议**实时计算**，减少定时任务依赖：

```java
private Integer computeTaskStatus(AttendanceTask task) {
    LocalDateTime now = LocalDateTime.now();
    if (now.isBefore(task.getStartTime())) return 0;
    if (now.isAfter(task.getEndTime())) return 2;
    return 1;
}
```

在 VO 转换时调用即可，不需要持久化状态字段。如需结束任务，可单独提供「结束任务」接口将 `status` 强制设为 2。

### 9.4 防止重复签到

数据库层已有唯一约束：

```sql
UNIQUE KEY uk_attendance_task_student (task_id, student_id)
```

业务层先查询再更新，避免触发数据库异常。也可依赖唯一约束捕获异常后返回友好提示。

## 10. 权限实现建议

当前 `LoginInterceptor` 为 TODO 状态，考勤模块开发时可按以下方式处理：

1. Controller 中从 Session 读取 `userId`、`role`、`relatedId`（学生ID或教师ID）。
2. 角色判断：
   - `STUDENT`：只能操作本人数据。
   - `TEACHER`：只能操作本人发布的任务数据。
   - `ADMIN`：可操作系统全部数据。
3. 所有权限校验在 Service 层完成，Controller 只做简单角色拦截。

示例工具方法：

```java
private void checkStudent(HttpSession session) {
    if (!"STUDENT".equals(getCurrentRole(session))) {
        throw new BusinessException(ResultCode.FORBIDDEN);
    }
}
```

## 15. 常见错误与规避

| 错误 | 原因 | 规避方法 |
| --- | --- | --- |
| 返回实体类给前端 | 暴露敏感字段 | 统一转换为 VO。 |
| 在 Controller 写业务判断 | 职责混乱 | 业务判断放到 Service。 |
| 忽略数据库唯一约束 | 高并发重复签到 | 预生成记录 + 唯一约束 + 业务层校验。 |
| 时间比较用字符串 | 精度丢失、时区错误 | 使用 `LocalDateTime` 比较。 |
| 前端传用户ID做权限判断 | 可被篡改 | 从 Session 获取当前用户。 |
| 硬编码数据库密码 | 安全风险 | 使用 `application-local.yml`，不提交到 Git。 |
| 直接物理删除考勤记录 | 数据不可追溯 | 不提供删除接口或仅管理员可标记失效。 |
