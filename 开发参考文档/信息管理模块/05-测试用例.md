# 信息管理模块 - 测试用例

## 一、认证模块

### 正常流程
| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| TC-AUTH-N-01 | 学生用正确学号和密码登录 | 返回200，session存入用户信息 |
| TC-AUTH-N-02 | 教师用正确工号和密码登录 | 返回200，跳转教师首页 |
| TC-AUTH-N-03 | 管理员用正确账号登录 | 返回200，role=ADMIN |
| TC-AUTH-N-04 | 退出登录 | session清除，返回登录页 |

### 异常流程
| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| TC-AUTH-E-01 | 用户名不存在 | 返回400，"用户名或密码错误" |
| TC-AUTH-E-02 | 密码错误 | 返回400，"用户名或密码错误" |
| TC-AUTH-E-03 | 账号已禁用 | 返回403，"账号已被禁用" |
| TC-AUTH-E-04 | 未登录访问需要登录的接口 | 返回401 |

## 二、公告管理

### 正常流程
| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| TC-NOTICE-N-01 | 管理员发布公告 | 返回200，公告已创建 |
| TC-NOTICE-N-02 | 发布公告带附件上传 | 返回200，附件保存到服务器 |
| TC-NOTICE-N-03 | 更新自己发布的公告 | 返回200，内容已更新 |
| TC-NOTICE-N-04 | 删除公告（软删除） | 返回200，is_deleted=1 |
| TC-NOTICE-N-05 | 分页查询公告列表 | 返回分页数据 |
| TC-NOTICE-N-06 | 切换置顶状态 | 返回200，top_flag取反 |
| TC-NOTICE-N-07 | 下载公告附件 | 返回文件流 |

### 异常流程
| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| TC-NOTICE-E-01 | 标题为空 | 返回400 |
| TC-NOTICE-E-02 | 内容为空 | 返回400 |
| TC-NOTICE-E-03 | 更新不存在的公告 | 返回404 |
| TC-NOTICE-E-04 | 下载不存在的附件 | 返回404 |

## 三、学生管理

| TC-STU-N-01 | 管理员新增学生 | 返回200，student+user_account各一条记录 |

| TC-STU-E-02 | 修改不存在的学生 | 返回404 |
